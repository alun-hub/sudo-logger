Audit-loggar som inte kan manipuleras

Standard sudo loggar på samma host som root — det är meningslöst. sudo-logger streamar varje knapptryckning i realtid till en isolerad server med mTLS, och om nätverket försvinner fryser kärnan sessionen inom 800 ms. Inga gap. Inga undantag.

Det dolda problemet med standard sudo

Root kan radera sina egna spår

De flesta sudo-verktyg loggar till samma host som användaren som kör sudo. Det innebär att en administratör med tillräckliga privilegier kan radera, stoppa eller kringgå loggningen. Det är inte ett audit — det är en illusion av kontroll.

🗑️

Loggarna ligger på samma disk

Standard sudo skriver till /var/log/auth.log eller auditd på samma host. Root kan ta bort dem, stoppa syslog, eller skriva över dem — innan din SIEM ens hunnit samla in dem.

rm -rf /var/log/auth* && systemctl stop rsyslog
📡

Nätverksavbrott = loggningsgap

Om SSH-anslutningen tappas mitt i en session — nätverksfel, timeout, omstart — fortsätter processen köra utan loggning. Du vet aldrig vad som hände under avbrottet.

Session fortsatte 14 min utan loggning efter nätverksfel
👻

Fail-open av design

Standardverktyg är designade för att låta sudo köra även om loggservern är nere. Loggning ses som "nice to have" — inte som ett krav. Det öppnar ett fönster som angripare kan utnyttja.

Loggserver nere i 3 min = 47 ologgade sudo-kommandon
Fyra garantier ingen annan ger

Designad för att vara omöjlig att kringgå

Varje säkerhetsegenskap är enforced i Linux-kärnan eller via mTLS-protokollet — inte av policyer i userspace som root kan stänga av.

01

Fail-closed — alltid

Om loggservern är otillgänglig när sudo startar avvisas sessionen innan kommandot exekveras. Inget undantag. Loggningen är ett krav, inte ett bästa-fall-scenario. En angripare kan inte vänta ut loggservern.

Ingen loggserver = ingen session. Period.
02

Kernel-freeze vid nätverksavbrott

Om ACK-heartbeat från servern uteblir i 800 ms fryser agenten alla processer i sessionen via cgroup.freeze — utan jobcontrol-signaler, så fg inte kan bryta ut. CLONE_NEWCGROUP-isolering hindrar även CAP_SYS_ADMIN från att migrera till ett annat cgroup och undkomma frysningen.

Freeze inom ~800 ms · automatisk återhämtning · timeout efter 3 min
03

20 kernel-hooks — root kan inte gömma sig

eBPF LSM Sandbox attachar 20 hooks i Linux-kärnan (16 LSM + 2 tracepoints). De blockerar skrivningar till skyddade filer, borttagning av sockets, bpf()-syscall, AF_NETLINK (brandväggsmanipulation), ptrace(), mount() och CAP_AUDIT_CONTROL — allt scoped per session-cgroup. Root kan inte byta ut agenten, döda auditd eller radera loggar.

Kernel-enforced · inte bypassbart ens av root
04

eBPF ring buffer — varje knapptryckning

Tre eBPF-tracepoints fångar TTY-I/O från alla processer i trackade cgroups — inte bara direkt under sudo. su, screen, tmux och pkexec fångas transparant inuti den omkringliggande sessionen. Om sudo kör utan plugin skapar divergence-detectorn ett synligt ⚠ no plugin-larm i replay-gränssnittet.

Ring buffer · sys_enter_write hook · divergence detection
Modulär uppbyggnad & Open-Core

Komplett plattform för privilegierad åtkomst

Från kernel-nivå härdning till centraliserad sudoers-hantering och SIEM-integration — allt i ett paket.

🔌
Sudo I/O Plugin OSS

Laddas direkt av sudo som en native I/O-plugin. Fångar stdin, stdout, stderr, tty-input, tty-output, miljövariabler och körda kommandon. Streamas i realtid — ingen lokal buffring på klienten. Plugin detekterar socket-drop inom 150 ms och skickar SIGTERM.

🛡️
eBPF LSM Sandbox Enterprise

20 kernel-hooks (16 LSM + 2 tracepoints) enforcar ett deny-list i Linux-kärnan. Blockerar skrivningar till skyddade filer och devices, kill-signaler till skyddade processer, bpf()-syscall, AF_NETLINK, ptrace, mount och farliga capabilities. Scoped per session via cgroup-ID och PID-tracking propagerat atomärt vid fork.

📡
mTLS + Ed25519 ACK-signering OSS

Klient och server autentiserar varandra med certifikat. Varje mottaget chunk bekräftas med en Ed25519-signerad ACK — agenten vägrar spara data som inte ACK:ats. Heartbeat skickas var 400 ms; sessionen fryses om ACK uteblir i 800 ms.

🔍
Automatisk hemlighetsredigering OSS

Agenten maskerar AWS-nycklar, API-tokens, Bearer-headers, JWT-tokens, URL-lösenord och andra hemligheter i terminalströmmen innan de når loggservern. Anpassade regex-mönster kan läggas till via mask_pattern i agent.conf.

⏳
Just-In-Time Godkännanden Enterprise

Kräv motivering och godkännande i realtid innan sudo-sessionen startar. Sessionen fryses i kärnan medan begäran väntar. Stödjer Mattermost/Slack med interaktiva Approve/Deny-knappar, tidsbegränsade godkännandefönster och automatisk TTL-stängning.

🧠
OPA JIT-policymotor Enterprise

Ersätt hårdkodade undantagslistor med ett fullständigt regelspråk baserat på Open Policy Agent och Rego. Regler kan matcha användare, host, kommando, OS-grupp (lokal/SSSD/LDAP/AD), veckodag och klockslag. Prioritet: deny > allow > challenge. Ändringar hot-reloadas utan omstart.

📋
Centraliserad Sudoers-hantering Enterprise

Push sudoers-regler till alla hanterade hostar från replay-gränssnittet. Agenten validerar och applicerar atomärt via visudo -c och rapporterar sync-status i realtid. Globalt default-template med per-host-override, visuell korteditor och raw-editor med syntaxmarkering.

☁️
S3 & PostgreSQL — skalbar HA Enterprise

Skala loggservrar horisontellt i Kubernetes. Sessioner lagras i S3-kompatibel objektlagring, metadata i PostgreSQL, med inbyggd leader election för SIEM-vidarebefordran. Designad för 500+ simultana sessioner med asynkron batch-disk-I/O och nolldeadlock-arkitektur.

📤
SIEM-integration OSS

Vidarebefordra risklarm i realtid till valfritt SIEM i JSON, CEF (ArcSight/QRadar) eller OCSF-format via HTTPS eller syslog. Konfigurationen hot-reloadas var 30:e sekund utan omstart. Prometheus-endpoint för Grafana-dashboards ingår.

🕵️
eBPF Divergence Detection OSS

Tre eBPF-tracepoints (TTY-I/O, execve, process-exit) jämför kernel-aktivitet med vad plugin rapporterat. Om sudo kör utan plugin visas ett ⚠ no plugin-larm i UI:t. Fångar även pkexec-körningar inuti sessioner och flaggar dem som separata händelser.

🚫
Central Block-policy Enterprise

Blockera specifika användare eller hostar omedelbart från centralt håll — ingen distribution av sudoers-ändringar till varje maskin. Agenten pollar blockeringslistan och avvisar nya sessioner för blockerade identiteter inom sekunder.

Hur det fungerar

Enkel driftsättning, stark säkerhet

En lättvikts plugin på varje host skickar krypterad sessionsdata till en central loggserver via mTLS. Inget behöver installeras på klienterna — sudo-integrationen sker transparent via /etc/sudo.conf.
