# AI Reliability & Integrity Mandates

To ensure code integrity and prevent syntactic/logic errors (such as 'n' prefixes or broken regex), the following mandates are in effect:

## 1. Surgical Editing Rule
- **NEVER** use `sed -i` for multi-line code injection.
- **ALWAYS** write code changes to a temporary file in `/tmp/` first.
- **ALWAYS** use the `replace` tool with at least 5-10 lines of context to ensure unique matching.

## 2. Mandatory Validation
- **GO**: Run `go fmt` and `go vet` after EVERY modification.
- **C**: Run `gcc -fsyntax-only -I/usr/include/sudo` after EVERY modification.
- **Verification**: You must report "Syntax check: PASSED" in your response.
