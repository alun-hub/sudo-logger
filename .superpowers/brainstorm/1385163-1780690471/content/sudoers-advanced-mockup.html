Sudoers Parameters We Will Map in UI

  • NOPASSWD — Execute commands without password prompts.
  • NOEXEC — Prevent executing subprocesses (essential for security).
  • SETENV — Let user specify environment variables.
  • CWD — Force the command to execute from a specific folder.
  • CHROOT — Sandbox the command in a chroot directory.
  • TIMEOUT — Avoid dangling sudo commands by setting a max runtime.

Kept in "Raw Advanced Rules" Textarea

  • Global Defaults (e.g. Defaults secure_path="...", Defaults mail_badpass).
  • Complex Aliases that aren't mapped to user cards (e.g. custom host groups, command groups).
  • Negated Rules (e.g. `!/bin/su`) which are fragile in UI and better written in raw syntax.
