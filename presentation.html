Varför räcker inte sudo?

Log-Manipulation

Standardloggar skrivs lokalt. En root-användare kan köra rm -rf /var/log/secure för att radera bevis på sabotage innan någon hinner reagera.

Nätverksluckor

Vid nätverksfel fortsätter sudo-sessionen ologgad. Administratören kan utföra kritiska ändringar som aldrig når central lagring.

Kernel Escapes

Processer kan flyttas av systemd eller "detacha" från terminalen. Vanlig I/O-loggning tappar då spåret helt.

Implicit Trust

Systemet litar blint på root. Inget hindrar root från att ändra i /etc/sudoers eller avaktivera säkerhetstjänster.

Vad händer om nätverket dör under en aktiv session?

  • 800ms: Timeout för ack-kvitton.
  • Freeze: Kärnan suspenderar cgroupen.

Zero Performance Hit

O(1) uppslag i kernel-hashmaps. Märks inte ens vid extrem I/O-belastning.

// eBPF LSM Hook Example
SEC("lsm/file_open")
int BPF_PROG(sandbox_open, struct file *f) {
  if (!in_sandbox()) return 0;

  // Block write-mode opens on protected inodes
  if (is_write_mode(f) && is_protected(f->inode)) {
    submit_alert(ALERT_FILE_OPEN);
    return -EPERM;
  }
  return 0;
}
